## [1.1.0] - 2026-02-23

### Добавлено
- **Парсинг деталей задач** — открытие AJAX-модалки для каждой задачи, извлечение:
  - URL для размещения (из `#copy_url` input)
  - Описание (текст задания + комментарий оптимизатора + анкор)
  - Требования к странице (тип обзора, внешние ссылки, индексация)
- **Telegram-уведомления** — отправка HTML-сообщений о новых задачах через Bot API:
  - Секция `[telegram]` в конфиге (`enabled`, `bot_token`, `chat_id`)
  - Форматирование с ценой, доменом, описанием, URL
  - Обрезка длинных сообщений до лимита Telegram (4096 символов)
- **Реальные тесты** — 58 тестов с assertions вместо заглушек:
  - Парсинг цен (10 форматов), извлечение task_id, парсинг строки задачи
  - Парсинг деталей (success, timeout, структура ответа)
  - Telegram: форматирование, HTML-экранирование, обрезка, отправка, ошибки
  - Очистка HTML (entities, пробелы)
- Поля `description`, `url`, `requirements`, `contacts`, `deadline` в SQL INSERT/UPDATE
- `insert_or_update_task()` возвращает статус: новая задача / обновлённая / ошибка

### Изменено
- `parse_task_row()` — title берётся из `.site-link__campaign` (тип задачи: Заметка/Контекстная ссылка) вместо текста "Подробнее..."
- `parse_task_list()` — после парсинга списка вызывает `parse_task_details()` для каждой задачи
- `main()` — отслеживает новые задачи, отправляет Telegram-уведомления
- `config.ini.example` — добавлена секция `[telegram]`

### Исправлено
- Модалка не закрывалась между задачами — добавлено принудительное удаление старых модалок через jQuery
- CSS-селекторы для модалки привязаны к реальной разметке сайта (`.tv_params_block`, `.block_title`, `.param`, `#copy_url`)

## [1.0.0] - 2026-02-05

### Добавлено
- Полная реализация парсера (~1091 LOC)
- Автоматическая авторизация на gogetlinks.net с поддержкой модального окна
- Решение reCAPTCHA через anti-captcha.com API (опционально)
- Парсинг списка задач: domain, customer, external_links, title, time_passed, price
- MySQL хранение с дедупликацией (UNIQUE INDEX на task_id, ON DUPLICATE KEY UPDATE)
- Структурированное логирование с ротацией (5MB, 3 бэкапа)
- Конфигурация из INI-файла с валидацией
- Exit codes для мониторинга (0=ok, 1=auth, 2=captcha, 3=config, 4=db, 5=webdriver, 99=unexpected)
- Makefile с 25+ командами автоматизации
- Полная SPARC документация (9 документов на 2 языках)
- Claude Code integration toolkit (agents, skills, rules)
- Структура тестов pytest (заглушки)
- Схема базы данных (`schema.sql`)
