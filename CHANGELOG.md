## [1.2.2] - 2026-03-05

### Добавлено
- **Lock для этапа `/mySites`** (`/tmp/gogetlinks_mysites.lock`) для защиты от параллельных запусков.
- Поддержка **stale-lock cleanup**: если lock старше 3 часов и PID мёртв — lock удаляется автоматически.
- Переменная окружения `GGL_SITES_LOCK_FILE` для переопределения пути lock-файла.

### Изменено
- При занятом mySites-lock скрипт выполняет **тихий пропуск** запуска (`exit 0`) с warning в логах.
- Lock применяется только к запускам, где выполняется `/mySites` (режим `--skip-sites` lock не использует).

### Тесты
- Добавлен `tests/test_locking.py` (acquire/release, stale lock, foreign lock, early exit в `main()`).
- Общее количество тестов: **84** (1 skipped).

## [1.2.1] - 2026-03-05

### Добавлено
- **Парсинг `/mySites`** с сохранением метрик в `ddl.domain` (`ggl_status`, `ggl_traffic`, `ggl_sqi`, `ggl_cf_tf`, `ggl_trust`, `ggl_update_at`)
- **Уведомления в Telegram о смене статуса сайта** (`ggl_status`) в том же боте/чате, без тегов `mention`
- **CLI-флаг `--skip-tasks`** для запуска только этапа `/mySites`

### Изменено
- **Разделение режимов cron-запуска:**
  - ежечасно: только задачи (`--skip-sites`)
  - ежедневно утром: только сайты (`--skip-tasks`)
- **Таблица задач переименована в `ddl.ggl_tasks`** (в коде и SQL-схеме)
- **`schema.sql` больше не создаёт базу данных**: база `ddl` должна существовать заранее
- **Парсинг `/mySites` проходит по всем страницам**, а не только по первой

### Исправлено
- **Anti-bot fallback при авторизации:** сначала direct, при блоке (`/403.php` / QRATOR) — повтор через proxy `127.0.0.1:3128`
- Убрано извлечение причины отказа для статуса `Отклонен, подробнее...` (сохраняется сам статус, без медленного обхода карточек отказа)

### Тесты
- Обновлены тесты базы/интеграции под новый контракт `save_sites_to_db()`
- Общее количество тестов: **77** (1 skipped)

### Ранее в ветке 1.2.1 (2026-02-24)

#### Исправлено
- **Парсинг цены** — заглавная кириллическая `Р` (U+0420) не удалялась регексом,
  содержавшим только строчную `р` (U+0440). Сайт gogetlinks возвращает цены в
  формате `'500 Р'` — `Decimal('500Р')` бросал `InvalidOperation` и тихо возвращал
  `0.00`. Добавлен флаг `re.IGNORECASE` в `parse_price()`.

#### Улучшено
- **Пропуск модалок уже сохранённых задач** — новая функция `task_has_details(conn, task_id)`
  проверяет наличие непустого `description` в БД. `parse_task_list()` теперь принимает
  `conn` и открывает AJAX-модалку только для задач без деталей. При 11 задачах в списке,
  где 10 уже в БД, время парсинга деталей снижается с ~17с до ~1.5с (экономия 1.5с на задачу).
- Добавлен `DEBUG`-лог сырого `price_text` перед парсингом для упрощения диагностики.

#### Тесты
- Добавлены тесты для форматов `'500 Р'` и `'0 Р'` в `TestPriceParser`
- Добавлен класс `TestTaskHasDetails` (6 тестов: hit, miss, cursor cleanup, параметризованный SQL)
- Общее количество тестов: **72** (было 66)

## [1.2.0] - 2026-02-23

### Добавлено
- **Сохранение cookie сессии** — пропуск полной авторизации при повторных запусках:
  - `save_cookies()` — сохранение cookies в `session_cookies.pkl` (pickle + chmod 600)
  - `load_cookies()` — загрузка cookies с проверкой прав файла и валидности сессии
  - Автоудаление протухшего cookie файла при expired session
  - 6 тестов для cookie функций (создание, сериализация, загрузка, expiry, permissions)

### Изменено
- `main()` — сначала пробует cached сессию, при неудаче — полная авторизация + сохранение cookies
- Общее количество тестов: 64 (было 58)

## [1.1.0] - 2026-02-23

### Добавлено
- **Парсинг деталей задач** — открытие AJAX-модалки для каждой задачи, извлечение:
  - URL для размещения (из `#copy_url` input)
  - Описание (текст задания + комментарий оптимизатора + анкор)
  - Требования к странице (тип обзора, внешние ссылки, индексация)
- **Telegram-уведомления** — отправка HTML-сообщений о новых задачах через Bot API:
  - Секция `[telegram]` в конфиге (`enabled`, `bot_token`, `chat_id`, `mention`)
  - Компактный формат: тип задачи | цена в ₽ | домен → клиент
  - Ссылка "Открыть задачи" на gogetlinks.net/webTask
  - Настраиваемые теги сотрудников через `mention` (например `@user1 @user2`)
  - Обрезка длинных сообщений до лимита Telegram (4096 символов)
- **Реальные тесты** — 58 тестов с assertions вместо заглушек:
  - Парсинг цен (10 форматов), извлечение task_id, парсинг строки задачи
  - Парсинг деталей (success, timeout, структура ответа)
  - Telegram: форматирование, HTML-экранирование, обрезка, отправка, ошибки
  - Очистка HTML (entities, пробелы)
- Поля `description`, `url`, `requirements`, `contacts`, `deadline` в SQL INSERT/UPDATE
- `insert_or_update_task()` возвращает статус: новая задача / обновлённая / ошибка

### Изменено
- `parse_task_row()` — title берётся из `.site-link__campaign` (тип задачи: Заметка/Контекстная ссылка) вместо текста "Подробнее..."
- `parse_task_list()` — после парсинга списка вызывает `parse_task_details()` для каждой задачи
- `main()` — отслеживает новые задачи, отправляет Telegram-уведомления
- `config.ini.example` — добавлена секция `[telegram]`

### Исправлено
- Модалка не закрывалась между задачами — добавлено принудительное удаление старых модалок через jQuery
- CSS-селекторы для модалки привязаны к реальной разметке сайта (`.tv_params_block`, `.block_title`, `.param`, `#copy_url`)

## [1.0.0] - 2026-02-05

### Добавлено
- Полная реализация парсера (~1091 LOC)
- Автоматическая авторизация на gogetlinks.net с поддержкой модального окна
- Решение reCAPTCHA через anti-captcha.com API (опционально)
- Парсинг списка задач: domain, customer, external_links, title, time_passed, price
- MySQL хранение с дедупликацией (UNIQUE INDEX на task_id, ON DUPLICATE KEY UPDATE)
- Структурированное логирование с ротацией (5MB, 3 бэкапа)
- Конфигурация из INI-файла с валидацией
- Exit codes для мониторинга (0=ok, 1=auth, 2=captcha, 3=config, 4=db, 5=webdriver, 99=unexpected)
- Makefile с 25+ командами автоматизации
- Полная SPARC документация (9 документов на 2 языках)
- Claude Code integration toolkit (agents, skills, rules)
- Структура тестов pytest (заглушки)
- Схема базы данных (`schema.sql`)
